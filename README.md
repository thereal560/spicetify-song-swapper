# Spicetify Song Swapper

Расширение для Spicetify, которое подменяет треки в очереди на локальные

## Зачем это нужно

Возможно, есть треки, версия которых на площадке Вас не устраивает, или которые невообразимым образом куда-то пропали (по не известной никому из нас причине). Данная приблуда даёт возможность слушать нужные вам треки, не меняя Ваших плейлистов.

## Как это работает

**Со стороны пользователя:**

0) [Установить Spicetify](https://spicetify.app/), установить это расширение
1) Включить в настройках отображение локальных файлов,
2) Закинуть в выбранную(-ые) папку(-и) нужные песни, проверить следующее:
	 - Если в метаданных файла указан автор и название песни - замечательно,
	 - В противном случае - добавить вручную или прописать название файла в формате "автор - название" (обязательно с пробелом, обязательно в этом порядке)
3) Перезагрузить Spotify.

**Со стороны кода:** 
При запуске Spotify получается список треков и их исполнителей из локальных файлов. При переключении трека в случае совпадения включается локальный трек.

## Скачать бесплатно

Вкладка с релизами, скачиваем .js файл, кладём его в:

 - Linux: `~/.config/spicetify/Extensions/`
 - Windows: `не помню`
 - MacOS: `не знаю`

В терминале/PowerShell прописываем `spicetify config extensions swapper.js`, потом `spicetify apply`.

## Известные проблемы/недочёты/ошибки/прочее
- Работает за счёт того, что добавляет локальный трек в очередь и сразу на него переключает (да, костыль, возможно, исправлю). Из-за этого:
	- Если в очереди уже есть треки, идущие как [заменяемый, незаменяемый, ...], заменённый трек ставится в конец этой очереди
	- При 2+ подходящих на замену треках, идущих друг за другом в очереди, заменяет все сразу, но потом воспроизводит в правильном порядке
		- Не получится включить предыдущий трек из этой очереди, только заново запустить текущий
- При включении заменяемого трека из поиска не создаётся очередь и этот трек играет по кругу
- ~Обновляет список локальных треков только один раз, при запуске -> добавить кнопку?~ Добавил кнопку: подождать, пока Spotify обработает изменения в папке -> библиотека -> ПКМ по Local Files -> Reload list of local tracks. 
- Проверяет только первого исполнителя из метаданных файла -> теоретически возможен супер странный маловероятный edge case при котором не будет совпадения не знаю даже фиксить нет
- Удалённые треки нельзя вручную добавлять в очередь -> включаются только если сами попадутся
